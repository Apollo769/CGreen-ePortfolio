Summary

The code review served as the starting point for the capstone project and provided an opportunity to evaluate the original A.I.M. (Agile Inventory Management) application. During the review, I analyzed the application's architecture, functionality, maintainability, and opportunities for improvement. The review helped identify strengths within the original artifact while also highlighting areas that could be enhanced to better demonstrate the Computer Science program outcomes.

Code Review Video

Original Artifact Overview

The selected artifact is A.I.M. (Agile Inventory Management), an Android inventory management application originally developed in CS 360: Mobile Architecture and Programming. The application allows users to create accounts, authenticate using Firebase Authentication, manage inventory items, update quantities, and receive low-stock notifications. Prior to enhancement, the application utilized a simpler architecture and local data management approach that provided opportunities for improvement in maintainability, scalability, and overall functionality.

Identified Areas for Enhancement

During the code review, three primary enhancement opportunities were identified. The first focused on improving software engineering and design through architectural refactoring, validation improvements, and increased maintainability using the Model-View-ViewModel (MVVM) pattern. The second focused on implementing algorithms and data structures to improve searching, sorting, filtering, and inventory management capabilities. The third focused on database improvements through the integration of Firebase Firestore to provide cloud-based data storage and user-specific inventory management.
