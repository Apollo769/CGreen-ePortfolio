Summary

The code review served as the starting point for the capstone project and provided an opportunity to evaluate the original A.I.M. (Agile Inventory Management) application. During the review, I analyzed the application's architecture, functionality, maintainability, and opportunities for improvement. Three primary enhancement opportunities were identified in the code reivew. The first focused on improving software engineering and design through architectural refactoring, validation improvements, and increased maintainability using the Model-View-ViewModel (MVVM) pattern. The second focused on implementing algorithms and data structures to improve searching, sorting, filtering, and inventory management capabilities. The third focused on database improvements through the integration of Firebase Firestore to provide cloud-based data storage and user-specific inventory management.
